    Add `processGalleryImage()` to the existing `ScanViewModel`. This function copies the gallery image URI content to a temp JPEG in cacheDir and updates the ViewModel state, then calls back with the path.

    Add to ScanViewModel.kt (inside the class, after the existing methods):

    ```kotlin
    /**
     * Copy a gallery image URI to a temp file in cacheDir.
     *
     * Runs on the calling coroutine scope (caller must be in a coroutine, e.g. LaunchedEffect).
     * On success, calls [onReady] with the absolute path of the temp file.
     * On error, calls [onError] with a description message.
     *
     * The content is read via ContentResolver — works for both content:// and file:// URIs
     * returned by ActivityResultContracts.GetContent.
     */
    fun processGalleryImage(
        context: android.content.Context,
        uri: android.net.Uri,
        onReady: (String) -> Unit,
        onError: (String) -> Unit = {}
    ) {
        _scanState.value = ScanState.Capturing
        try {
            val tempFile = java.io.File(
                context.cacheDir,
                "temp_gallery_${System.currentTimeMillis()}.jpg"
            )
            context.contentResolver.openInputStream(uri)?.use { input ->
                tempFile.outputStream().use { output ->
                    input.copyTo(output)
                }
            } ?: run {
                _scanState.value = ScanState.Error("Could not open gallery image")
                onError("Could not open gallery image")
                return
            }
            tempImagePath = tempFile.absolutePath
            _scanState.value = ScanState.Idle
            onReady(tempFile.absolutePath)
        } catch (e: Exception) {
            _scanState.value = ScanState.Error(e.message ?: "Gallery import failed")
            onError(e.message ?: "Gallery import failed")
        }
    }
    ```

    Add required imports at the top of the file (if not already present):
    ```kotlin
    // Already present from Android SDK transitive — no explicit import needed for android.content.Context
    // and android.net.Uri if they are in the same module. Add if the compiler requires them:
    // import android.content.Context
    // import android.net.Uri
    ```

    NOTE: `processGalleryImage` is NOT a suspend function. It runs synchronously on whatever thread calls it. This is intentional — `ContentResolver.openInputStream` and `copyTo` are fast for typical business card images (< 5 MB). The ViewModel updates `_scanState` to `Capturing` at start and `Idle` on success, matching the camera capture pattern. If performance is a concern in practice, wrap in `viewModelScope.launch(Dispatchers.IO)` in Phase 3+.
